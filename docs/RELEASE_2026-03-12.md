# TillFlow Release Summary — 2026-03-12

## Overview

This release moves TillFlow into a cleaner production state across four key areas:

- local developer reliability on Windows/OneDrive
- POS speed and next-customer readiness
- report consistency and owner-facing clarity
- Ghana-first defaults, copy, and demo realism

## Production deployment

- Production alias: https://supermarket-pos-five.vercel.app
- Successful production deployment: https://supermarket-27pwx1z9w-joshua-owusus-projects.vercel.app
- Deployment method: manual Vercel production deploy from repository root

## Main release goals

### 1. Local development reliability

Improved the local startup flow so `npm run dev` is more resilient in this workspace.

Included changes:

- tolerate locked Prisma Windows engine DLL refreshes when an existing client is already present
- reuse an already-running TillFlow dev server instead of spawning duplicates
- fall back across ports `6200` to `6204`
- print a clear local URL to open
- allow local server-action origins across the fallback dev port range

## 2. POS flow improvements

Refined the protected POS experience to feel faster and clearer for frontline cashiers.

Included changes:

- faster reset into a next-customer-ready state after checkout
- extracted POS logic into reusable hooks and helper modules
- improved parked-cart handling, recall flow, and parked-sale visibility
- improved barcode and product-search handling
- manual MoMo remains available for real-world reconciliation workflows
- sticky desktop summary behavior fixed
- added focused POS and helper tests

## 3. Reports and owner insights

Reduced duplication across reporting pages and improved consistency in filters, summaries, and owner-facing views.

Included changes:

- shared report UI components for filters, actions, summaries, section headers, and tables
- shared report date parsing helpers
- shared store selection helpers
- shared operational metrics helpers for inventory and receivables
- improved cash drawer, cashflow, dashboard, income statement, margins, reorder suggestions, risk monitor, and owner pages
- tightened owner dashboard inventory typing and risk rendering

## 4. Ghana-first market-fit updates

Aligned more of the product with Ghanaian and broader African retail expectations.

Included changes:

- default currency alignment toward `GHS`
- `GH₵` / Ghana-first pricing shown in welcome-page preview content
- settings copy and currency guidance improved
- onboarding and registration copy localized more appropriately
- demo customers and supplier naming made more realistic
- supplier fallback wording improved
- receipts updated with cleaner customer-facing closing copy
- export/report metadata now includes currency context
